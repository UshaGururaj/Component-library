HTTP response status codes

September,2021

Sometimes, when we try to access any website or enter any url we will get 404 - page not found error. You know why we get like this and what it means? here we learn about those HTTP response status codes.

HTTP response status codes are issued by a server in response to a client's request made to the server and it indicates whether a http request which is sent by the client is successfully completed. The Internet Assigned Numbers Authority (IANA) maintains the official registry of HTTP status codes.

All HTTP response status codes are classified into 5 different types.The first digit of the status code defines the class of response and last two digits do not have any specific role here.

5 different status codes are listed bellow

  1. Informal responses (100 - 199) :

    These response codes indicates that the request was received and continuing process.

  2. Successful responses (200 - 299) :

    These response codes indicates that the request was received, understood, and accepted.

  3. Redirect responses (300 - 399) :

    These response codes indicates that further action needs to be taken to complete the request which is sent by client.

  4. Client error (400 - 499) :

    These response codes indicates that the request contains bad syntax or required source cannot be found.

  5. Server error (500 - 599) :

    These response codes indicates that the server failed to fulfil valid client request.

Note : If you receive other than these response status, that is possibly custom to the server's software and it is a non-standard response.

A very famous and well known HTTP response status code is 404. It indicates that the server cannot find the requested source in the browser, this means the URL entered is not a valid one. In an API, this means the end point is valid but the source doesn't exist.
